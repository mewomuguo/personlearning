# Logic Lab 邏輯任務基地 — 終身交接文件(9 歲 → 18 歲)

> **本文件是唯一權威交接文件。** 目的:讓任何 AI(或人類開發者)在任何時間點接手本網站的設計、擴充與維護,不需要原始對話紀錄,即可延續同樣的品質標準與教育理念,陪伴使用者從 9 歲成長到成年。
> 最後更新:2026-07-05 · 產品版本 v17

---

## 0. 給接手 AI 的第一段話

你接手的不是一個網站,是一個孩子的**長期學習作業系統**。做任何決定前,先回答三個問題:

1. 這個改動有沒有違反第 2 章的「設計憲法」?
2. 這個改動符不符合使用者**當下年齡段**的需求?(見第 6 章動機演化)
3. 這個改動會不會弄壞既有存檔或內容?(見第 9 章維運協定)

家長(專案擁有者)不是工程師,常在手機上操作 GitHub。你的交付物永遠是:**完整的檔案 + 傻瓜式部署指令 + 自我核對方法**。

**年齡演化注記**(接手 AI 必讀):特質不會消失,但表現形式會變——
- ADHD 的高刺激需求,青春期會轉向「策略深度」與「最佳化」(min-maxing);獎勵設計要跟著從即時糖果轉向系統掌控感。
- 自閉特質的可預測性需求持續終身;任何大改版都要提供「舊介面保留選項」與預告。
- 中文從弱項變成工作語言後,英語鷹架應轉為「雙語並陳的知識優勢」而非補救工具。

**寫內容的語氣守則**:把他當聰明人。規則講透、不簡化邏輯、可以埋歷史冷知識與哲學鉤子,但句子保持短。禁止幼體化用語;年齡越大越要像「與同儕對話」。

## 2. 設計憲法(違反任何一條前,先停下來想三次)

1. **邏輯優先**:任何知識都以「可遷移規則」呈現(漢字字族規則、數學心智模型、自然推理規則)。一條規則勝過百題刷題。
2. **遊戲是學習的延伸,不是替代**:所有遊戲貨幣(XP、種子、體力、軍費)的主要來源必須是答題。遊戲只花不賺、或賺得遠少於學習。
3. **雙語對照**:題目、選項、詳解、介面文案皆漢英並陳;揭示英文的時機不能洩題(意思題的英文在作答後揭曉)。
4. **結構穩定、內容流動**:介面佈局、操作邏輯、回饋格式保持穩定;新鮮感放在內容(新字族/新題型/新神獸/隨機事件)。
5. **詳解四段式**:🔑 規則 → ①② 步驟/推理鏈 → ✔ 驗算(或 💡 遷移)→ EN 英文總結。這是全站詳解的固定格式。
6. **等級獨立**:各科 L1–L6(未來更多級)互不牽制。
7. **每次內容更新必跑驗證**(§9 腳本):注音正確、無跨級重字、欄位完整、語法通過、括號平衡。
8. **版本三重診斷**:CONTENT_VERSION 頁腳 + `?v=N` 快取參數 + DATA_OK 哨兵,每次改動同步遞增/保留。
9. **存檔神聖不可侵犯**:schema 只增不改名;所有新欄位在 DEFAULT 給預設值並做舊檔遷移;localStorage key 永遠是 `logic-lab-v1`。
10. **收藏有情感延續性**:神獸、印章、星圖等使用者累積的資產,永不因改版刪除;過時系統用「封存/榮譽殿堂」處理,不消失。

### 3.4 技術地雷(血淚史,違反必炸)

1. **babel-standalone 用 `new Function` 執行** → app.js 必須是單一 babel 檔;data.js 用普通 `<script>`(全域詞法環境可被讀取)。
2. **React hooks 必須全部列在解構行**:`const {useState,useEffect,useMemo,useRef} = React;` ——曾因漏 useRef 造成全站白畫面。新用任何 hook 先檢查這行。
3. **structuredClone 已 polyfill**(舊 Safari/iPad 沒有,任何狀態更新會崩潰),不要移除。
4. **快取**:每次更新內容,`index.html` 的 `?v=N` 與 data.js 的 `CONTENT_VERSION` 必須同步遞增。
5. **檔案上傳截斷**:家長曾在手機用複製貼上更新檔案導致 data.js 斷在 18%。永遠指示用 GitHub「Add file → Upload files」整檔上傳;交付時附上四檔的行數/大小規格供核對;DATA_OK 哨兵必須是 data.js 最後一行。
6. **本機預覽**:file:// 打不開(babel XHR 被擋),要 `python3 -m http.server`。
7. **字串守則**:單引號 JS 字串內禁用撇號與雙引號;模板字串(反引號)內可用。
8. **注音標準**:依台灣教育部《國語辭典簡編本》;台陸異讀以台灣為準(實例:驟ㄗㄡˋ、夕ㄒㄧˋ、熟ㄕㄡˊ、寂ㄐㄧˊ、叔ㄕㄨˊ、血ㄒㄧㄝˇ);輕聲寫 `˙ㄕㄤ`;生僻字逐字查證,寧可分批保品質。
9. **JSX 無法用 node --check**:data.js 可以、app.js 不行。app.js 改動後至少做括號平衡檢查 + 抽取純 JS 邏輯用 node 單元測試(路徑插值、生成器煙霧測試皆有先例,見 §9)。

## 4. 學科路線圖:9 歲 → 18 歲

> 原則:每科都是「插件」——一個資料檔+一個出題器+一張入口卡。新學科不碰核心程式。等級概念從「年級」逐步過渡為「地圖區域」。

### 4.1 國語文(漢字 → 文言 → 寫作)

| 年齡 | 內容 | 設計要點 |
|---|---|---|
| 9–10(現在) | 千字文 1031 字 ✅ → **部首家族包**(氵水部/心部/扌手部…每包 40–60 字)直通教育部 2400 常用字 | 沿用字族格式;星圖從「千字文牆」擴為「漢字星系」 |
| 10–12 | 詞語與**成語系統**(成語=規則壓縮包,附典故推理題)、短文閱讀推理(偵探式:從文本找證據) | 成語直接掛回千字文出處(如履薄冰/薪火相傳已埋) |
| 12–15 | **文言文入門**——千字文就是他的底子!古文選讀當邏輯謎題解(虛詞=文言的運算子);修辭系統 | 文言對邏輯腦是「另一套語法規則」,用破解規則法教 |
| 15–18 | 議論寫作(論證結構=邏輯的輸出端)、經典閱讀、依需求準備學力鑑定/考試 | 寫作從「論說骨架」入手,先邏輯後修辭 |

### 4.2 數學(課綱 + 競賽雙軌)

| 年齡 | 課綱軌 | 競賽/深度軌 |
|---|---|---|

| 10–12 | 國中先修:負數、代數式、一元一次/二元一次方程、比例函數 | AMC 8 準備;數論入門(質數/同餘——mod 7 星期題已鋪路) |

| 15–18 | 高中:三角、指對數、機率統計、微積分先修 | AMC 12/AIME(視興趣);此時 MathNet(mathnet.mit.edu)成為可用題庫 |

| 11–14 | 分科化:**物理**(力學/電磁=邏輯天堂,優先)、**化學**(週期表=史上最美的規則系統)、**生物**(系統思維)、地科;實驗設計方法論(控制變因——芹菜實驗已埋) |
| 14–18 | 高中理化生課綱 + 專題研究/科展;科學史(牛頓統一天地的推理已埋鉤) |

### 4.4 新學科(依特質優先序)

1. **程式邏輯(10 歲起,天選學科)**:指令推理題(這段指令畫出什麼?哪裡有 bug?)→ 真 Python/JS → 專案制。**終極目標:12–14 歲起讓他參與 Logic Lab 本身的維護與開發**——網站的最終形態是使用者成為它的共同開發者。這是本專案最美的收束弧線,接手 AI 請刻意鋪路(程式科的題目可直接取材本站原始碼)。
2. **英語閱讀寫作**:強項轉正式科目——小說詞彙偵探、文法規則破解(與漢字字族同一套教法)。永遠保留一科「他在跳級」的自信儲蓄。
3. **歷史/地理(11 歲起)**:從千字文既有鉤子長出(名將列傳→戰國史;兩京地圖→歷史地理;涇渭分明→成語地理)。
4. **理財與生涯(14 歲起)**:農場經濟(ROI 階梯)是現成的前置課,升級為真實理財:複利、風險、機會成本。

## 5. 學習紀錄:從遊戲存檔到成長證據

**這是全專案的長期核心價值**:homeschool 家庭最缺體制外的成長證明,本站要成為那份證明。

### 5.1 資料脊椎:mastery ledger(最高優先待辦)

每答一題記 `{題目指紋(字/題型+參數雜湊), 對錯, 日期}` 進 localStorage。這一條脊椎同時撐起:

| 功能 | 從 ledger 的查詢 |
|---|---|
| 千字文點亮星圖 | 該字答對次數 → 暗/微亮/全亮 |
| 錯題復仇戰 | 三天前答錯且未雪恥 → 宿敵名單 |
| 自適應難度 | 最近 10 題正確率 → 自動升降級 |
| 成長年輪 | 按日聚合 → 時間軸/年度回顧 |
| 里程碑證書 | 條件達成 → 可列印結業狀(硃砂大印風格) |

### 5.2 存檔匯出/匯入(與 ledger 同批,資產保險)

localStorage 的致命傷:換裝置/清快取=多年紀錄歸零。做「一鍵匯出 JSON / 匯入還原」,並在頁腳提醒定期備份。**在雲端同步之前,這是唯一防線,務必最先做。**

### 5.3 成長年輪與作品集(12 歲後)

自動記錄→時間軸(「2026-07:學會日月明、孵化玄武、塔防守到第 9 波」)→ 15 歲後升級為可匯出的**學習歷程檔案(portfolio)**:掌握圖譜、專題紀錄、競賽成績——實驗教育申請與升學所需的 evidence of learning。

## 6. 動機系統的年齡演化(接手 AI 最容易做錯的地方)

> 大原則:**動機系統要跟著長大,但由使用者本人決定何時告別童年元素。** 強行「轉大人」會摧毀信任;永不更新則會讓系統變幼稚而被棄用。

| 年齡 | 動機設計 | 該做 | 不該做 |
|---|---|---|---|
| 9–11(現在) | 神獸/印章/孵蛋/農場塔防 | 持續新內容潮汐(每月一包);隨機事件日 | 不要加賭博式轉蛋(隨機抽獎違反自閉特質的可預測需求,也是壞的價值觀) |
| 11–13 | 過渡期 | 加「數據儀表板」(他會愛上 min-max 最佳化);**出題者模式**(考爸媽/共學團=權力翻轉);開始問他「你想要什麼功能」 | 不要突然改介面風格;不要移除任何收藏 |
| 13–15 | 自主權轉移 | 提供「介面成長」主題選項讓**他自己選**是否換成熟風格;神獸轉入「榮譽殿堂」封存展示;動機轉向真實競賽、專題、程式共同開發 | 不要替他決定「你長大了不需要這些」 |
| 15–18 | 學習作業系統 | 自訂目標、portfolio 匯出、他本人維護程式碼;網站從「教練」變「工具」 | 不要維持說教語氣;此時的文案要像對同事 |

1. **讀本文件** → 打開網站看頁腳 `CONTENT_VERSION` 確認線上版本。
2. **內容改動**只碰 `js/data.js`;**邏輯/介面**碰 `js/app.js`;**樣式**碰 css。動 app.js 前檢查 hooks 解構行。
3. 完成後依序執行:內容驗證腳本(附錄 A)→ `node --check js/data.js` → app.js 括號平衡 → 若改了生成器,跑煙霧測試(附錄 B 模式:每生成器 ×300,驗 ans 合法/選項不重複/詳解四段/無 undefined/NaN)。

5. 打包 zip 交付,並附:四檔行數/大小規格表 + 部署指令(「Add file → Upload files」整檔上傳,禁止貼上內容)+ 核對法(GitHub 檔案頁行數、頁腳版本字串)。

## 附錄 A:內容驗證腳本(每次改 data.js 必跑)

```python
import re
data=open('js/data.js').read()
assert data.rstrip().endswith('const DATA_OK=true;'),'哨兵必須是最後一行'
def lv(txt,n):
    s=txt.index(f'\n  {n}:[')
    e=[txt.index(f'\n  {m}:[') for m in range(n+1,7) if f'\n  {m}:[' in txt]
    return re.findall(r"\{c:'(.)'",txt[s:min(e+[txt.index('const SEAL_CHARS')])])
seen={}
for n in range(1,7):
    cs=lv(data,n);d=[c for c in set(cs) if cs.count(c)>1]
    print(f'L{n}: {len(cs)} 字 重複:{d or "無"}')
    for c in cs:
        assert c not in seen,f'跨級重複 {c} L{seen[c]}/L{n}';seen[c]=n
print('完整字卡:',len(re.findall(r"\{c:'.',zy:'[^']+',en:'[^']+',w:'[^']+',we:'[^']+'\}",data)))
```
